## How It Works Now

### Request Lifecycle

```
1. User visits: https://your-site.netlify.app/services

2. Netlify CDN receives request
   вҶ“
3. netlify.toml redirect rule:
   from = "/*"
   to = "/.netlify/functions/ssr"
   status = 200 (silent rewrite)
   вҶ“
4. netlify/functions/ssr.ts handler executes
   в”ңв”Җ Receives Netlify event
   в”ңв”Җ Converts to Web Request API
   в”ңв”Җ Imports @tanstack/react-start/server-entry
   в”ңв”Җ Calls serverEntry.fetch(request)
   в””в”Җ Returns rendered HTML
   вҶ“
5. TanStack Start renders page
   в”ңв”Җ Loads route matching /services
   в”ңв”Җ Renders React components
   в”ңв”Җ Includes hydration data
   в””в”Җ Streams HTML response
   вҶ“
6. Browser receives HTML
   в”ңв”Җ Renders page immediately
   в”ңв”Җ Loads React bundle
   в”ңв”Җ Hydrates components
   в””в”Җ App becomes interactive
   вҶ“
7. User sees fully rendered page вң…
   (No 404 even on page refresh!)
```

## Performance Metrics

### Build Size
- **Client Bundle**: ~350 KB (gzipped: ~110 KB)
- **Static Assets**: ~6.5 MB (images, fonts)
- **Server Bundle**: Minimal (only for cold start)

### Runtime Performance
- **Cold Start**: ~100-500ms (Netlify Functions)
- **Warm Start**: ~10-50ms
- **Page Load**: ~500-1000ms typical (depends on network)
- **Time to Interactive**: ~1-2s (with hydration)

### Caching
- **Static Assets**: Cached for 1 year (immutable)
- **HTML**: Always revalidated (cache-busting)
- **CDN**: Global Netlify edge locations
